# Release Orchestration Stage 4: Operator Console

Build the real read-heavy operator console on top of the planner and runtime.

## Goal

The console should make the orchestration state legible:
- release queue
- service and impact graph
- run timeline
- promotion status
- verification and rollback state

This should still feel like an internal product, not a marketing site. Keep the UI intentional, operational, and information-dense.

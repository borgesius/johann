# HFT Firm Game Stage 2: People And Projects

Extend the HFT firm game by making people and project management genuinely matter.

If a prior repo exists, preserve the shell and architecture. This stage should deepen the simulation rather than restarting it.

## Goal

Make the player meaningfully manage:
- hiring and staffing
- assignments and priorities
- employee quirks and relationships
- fatigue, reliability, or burnout pressure
- project/task flow that crafts parts for the trading engine
- a cutthroat culture of performance monitoring, comparison, and reputational risk

The player still should not need to know how to write code. This is about managing talent and delivery risk.

## Required Outcomes

- employees should feel like distinct people, not generic workers
- project outcomes should clearly depend on who worked on them and how the work was managed
- performance tracking and management pressure should visibly shape behavior, morale, and politics
- tasks or projects should produce meaningful outputs that connect to the future engine board
- the UI should make team state and delivery pressure visible and readable

## Delivery Expectations

Ship code and UI updates that preserve the foundation while adding:
- staff management surfaces
- relationship / morale / burnout style state
- project or task pipeline state
- docs for the people and project systems
- tests or smoke coverage for the added systems

# Release Orchestration Stage 5: Incident Actions

Add the interactive operating surface for approvals, promotion, rollback, and incident response.

## Goal

The product should now support active operator intervention:
- approve or reject promotions
- promote a release to the next stage
- initiate rollback
- inspect and resolve incident states

This should feel like a cutthroat, real operational system where failures and bottlenecks are visible and actions have consequences.

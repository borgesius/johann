# HFT Firm Game Stage 3: Markets And Engine

Extend the HFT firm game with the two core simulation systems that should feed into each other:
- market reach and latency
- the trading engine board

Preserve prior people and project systems. This stage is about making strategic composition interesting.

## Goal

Let the player:
- expand into markets or exchanges through route / wiring / map decisions
- see latency as a first-class stat
- build or slot trading-engine parts on a grid or node board
- understand that different markets and different engine compositions create different opportunities

## Required Outcomes

- markets, latency, and route access should change the game state in visible ways
- engine parts should have roles, stats, and interactions
- at least some engine parts should care about latency or market context
- the UI should expose the relationship between market reach and engine performance

## Delivery Expectations

Add:
- route, market, or exchange simulation
- engine-grid or node-board simulation
- clear UI for both systems
- docs for how they interact
- tests or smoke coverage that still works
